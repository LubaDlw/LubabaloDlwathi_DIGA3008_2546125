BLOG WEEK 3

Planning and organizing my portfolio website with focus on structure, user experience, and information architecture.

My Goals for My Website

I want my portfolio website to showcase my diverse skills in game design, software development, and visual design work. My main goal is to present my projects and creations in a way that highlights my unique style and technical abilities. I need to develop intuitive navigation that allows visitors to explore my different design domains seamlessly. My site will feature interactive demonstrations of my game prototypes, code samples, and visual design work. I aim to guide potential clients or employers through my professional journey, showing the evolution of my skills and creative approach.

My Target User Analysis

My website primarily serves potential employers, clients, and fellow designers who are interested in my game, software, and visual design capabilities. These visitors have specific needs: quick access to my project highlights and technical skills; interactive examples that demonstrate my game mechanics and visual style; practical demonstrations of my problem-solving approach; and a portfolio organized to showcase both my specialized skills and versatility across different design disciplines.

My Content Mapping

My Information Structure

I'll organize my content into these main categories reflecting my expertise areas:

  • Game Design Portfolio
    • Game concepts and mechanics
    • Development process documentation
    • Link to itch.io games published
  • Software Development Projects
    • Web applications
    • Link to actual websites
  • Visual Design Work
    • UI/UX design projects
    • Illustrations and concept art
    • Brand identity projects
  • Blog Page
    • Make a book that the user can navigate to read blogs

My Information Architecture (IA)

Following the principles from the slides, I'll organize my content hierarchically:

  • Primary navigation for main categories (Games, Software, Visual, Blog)
  • Secondary navigation within each category for projects/subcategories
  • Card sorting with potential users to validate my categorization
  • Clear labeling system using terminology familiar to my target audience

My User Experience Flow

User Journey Mapping

My portfolio flow begins with an engaging introduction that showcases my best work across all disciplines. From there:

  • Landing Page: Featuring my strongest and latest projects
  • Portfolio Navigation: Allowing direct access to game, software, or visual design sections
  • Project Deep Dives: Detailed case studies of my process and problem-solving to show how i think and approach my designs
  • Contact and Resume: Professional information

My Interface Elements

Navigation Components

Following the slides' categories, I'll implement:

  • Primary Navigation: Main menu with clear icons for Games, Software, Visual, and Blog sections
  • Breadcrumbs: Showing current location (e.g., Home > Games > Project Name)
  • Pagination: For my blog "book" interface to create an immersive reading experience

Input Controls

  • Buttons: Clearly labeled call-to-action buttons for exploring projects
  • Toggles: For switching between grid/list views of projects
  • Interactive Elements: Direct manipulation of game demos and prototypes

Informational Components

  • Tooltips: For explaining technical terms or providing context
  • Message Boxes: For form submission confirmation

My Progress

Looking back at my development progress so far, I'm making steady headway on my portfolio website. I've established the foundation by implementing the HTML structure, which has given me a solid semantic framework to build upon. This initial markup phase has been crucial for organizing my content logically and ensuring accessibility from the start.

I've also begun incorporating images into my portfolio, which is bringing my work to life visually. Seeing my game designs, software projects, and visual art start to appear on the page has been rewarding, though I still need to optimize these assets for web performance.

One of my main focus areas has been improving my file structure. I've reorganized my project files into more logical directories that reflect the content categories I mapped out (games, software, visual design, and blog). This cleaner organization will make it much easier to maintain the site as I add more projects and blog content over time.
